Fix: Correct fix-imports.js regex to preserve quote types in imports

The fix-imports.js script had a regex bug where it hardcoded double quotes
in the replacement, causing mismatched quotes in compiled output like:
  from 'path.js"  (WRONG - single quote open, double quote close)

The fix uses a backreference pattern to preserve the original quote type:
  from 'path.js'  (CORRECT - same quote type)
  from "path.js"  (CORRECT - same quote type)

This ensures all compiled files have properly formatted imports.

- node --check confirms no syntax errors
